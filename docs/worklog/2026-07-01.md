## 팁(알아두면 좋은 정보)

- **로컬 백엔드 구동에는 JDK 17이 필요**함(`build.gradle`의 toolchain이 17 고정). 이 컨테이너엔 JDK 21만 있어서 `apt-get install openjdk-17-jdk-headless`로 별도 설치하고 `JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64`을 gradle 명령에 지정해야 함. `apt-get update` 먼저 안 하면 특정 버전 404가 날 수 있음.
- **로컬 Postgres**: `postgresql-16` 패키지는 이미 깔려 있지만 서비스가 꺼져 있음 → `pg_ctlcluster 16 main start`로 기동, `sudo -u postgres psql`로 `sheet_user`/`sheet_music` DB를 애플리케이션 기본값과 맞춰 생성.
- **여러 브랜치를 오가며 로컬 DB를 재사용하면 Flyway 마이그레이션 버전 충돌이 남**(예: 한 브랜치는 V11이 `users` 테이블, 다른 브랜치는 V11이 `performance_key` 컬럼). 증상은 "Migration checksum mismatch for migration version 11". 해결은 `DROP DATABASE`/`CREATE DATABASE`로 로컬 DB를 매번 깨끗하게 재생성하는 것이 제일 빠름 — 로컬 테스트 데이터는 어차피 휘발성이라 문제 없음.
- **`frontend/.env.local`로 로컬 백엔드(`http://127.0.0.1:8080`)를 가리키게 하고 테스트한 뒤, 끝나면 반드시 삭제**할 것(안 그러면 다음 세션이나 배포 설정에 영향 줄 수 있음). CLAUDE.md 기본 안내는 배포된 백엔드(`https://worship-sheet.fly.dev`)를 가리키게 되어 있지만, 이 원격 컨테이너의 네트워크 정책상 외부 `.fly.dev` 도메인이 프록시에서 막혀 있어서(403) 로컬 백엔드를 직접 띄워야 했음.
- **Playwright로 스타일러스/터치를 흉내낼 때**: `new TouchEvent(...)`에 넘기는 `changedTouches`는 반드시 `new Touch({...})`로 만든 실제 `Touch` 객체여야 함(plain object를 넣으면 `Failed to convert value to 'Touch'` 에러). 펜 입력은 `new PointerEvent(type, { pointerType: 'pen', pressure, ... })`으로 직접 dispatch하면 필압까지 재현 가능.
- **pkill은 이 컨테이너에서 예상과 다르게 동작(불필요하게 명령 전체가 종료됨)할 때가 있었음** — `ps aux`로 정확한 PID를 확인한 뒤 `kill -9 <pid...>`로 콕 집어 종료하는 편이 안전함.
- **GitHub PR 구독(`subscribe_pr_activity`) 이후 오는 Vercel 봇 코멘트(Building/Ready 상태 갱신)는 대부분 그냥 상태 알림**이라 별도 조치 없이 넘어가면 됨. 실제 리뷰 코멘트나 CI 실패와는 구분해서 대응할 것.
- **병렬로 여러 기능 브랜치에 걸쳐 DB 마이그레이션을 추가해야 한다면, 버전 번호를 미리 조율하거나(예: 이슈 번호 순으로 예약) PR 설명에 충돌 가능성을 명시**해두는 게 병합 시점 혼란을 줄임. 이번엔 #77·#75가 둘 다 V11을 써서 충돌이 예정되어 있고, #109는 의도적으로 V12를 선택해 최소 하나의 충돌을 피함.
